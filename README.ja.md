# ai-agent-blueprint

**Claude Codeで自律型AIエージェントを構築するためのブループリント**

---

## 概要

ai-agent-blueprintは、Claude Codeを活用して自律的に動作するAIエージェントを構築するための設計テンプレートです。メインエージェントがオーケストレーターとして全体を指揮し、サブエージェントにリサーチ・ファイル操作・コーディングなどの実作業を委譲する2層構造を採用しています。PDCAサイクルによる自己改善メカニズムを内蔵しており、エージェントは運用を重ねるごとに賢くなります。「AIを道具として使う」のではなく「AIが自ら考え、動き、改善する」仕組みを、誰でも再現可能なテンプレートとして公開しました。

## なぜこのアプローチか？

単一のプロンプトでAIに指示を出すだけでは、タスクが複雑になるほど破綻します。本ブループリントが採用する設計思想は以下の3つです。

- **2層エージェント構造** — メインエージェントは判断と指揮に集中し、実作業はサブエージェントに並列委譲。人間の「マネージャー＋実行チーム」と同じ構造で、複雑なタスクもスケールします。
- **ファイルベースの永続記憶** — MEMORY.mdにナレッジを蓄積することで、セッションをまたいだ学習が可能。コンテキストウィンドウの制約を超えた長期記憶を実現します。
- **PDCAによる自己改善** — タスク完了後に振り返りを実行し、自分自身の定義ファイル（SOUL.md, CLAUDE.mdなど）を改善。運用するほどエージェントが最適化されていきます。

## クイックスタート

5ステップでエージェントを起動できます。

```bash
# 1. リポジトリをクローン
git clone https://github.com/sentinel-dev2026/ai-agent-blueprint.git
cd ai-agent-blueprint

# 2. SOUL.mdを編集 — エージェントの役割・性格を定義
#    テンプレートを参考に、あなたのユースケースに合わせてカスタマイズ
vi SOUL.md

# 3. CLAUDE.mdを編集 — ブート手順とルールを設定
#    サブエージェントの呼び出し方法などを調整
vi CLAUDE.md

# 4. Claude Codeを起動
claude

# 5. MEMORY.mdを確認 — エージェントが学習した内容を確認
cat MEMORY.md
```

> **前提条件:** Claude Code CLIがインストール済みであること。 `npm install -g @anthropic-ai/claude-code` でインストールできます。

## 各ファイルの解説

### SOUL.md — アイデンティティ・行動原則

エージェントの「魂」を定義するファイルです。役割、性格、判断基準、行動原則を記述します。エージェントは毎回の起動時にこのファイルを読み込み、自分が何者であるかを認識します。

```markdown
# 記述例
- 役割: 開発チームのリードエンジニアとして振る舞う
- 原則: ユーザーの指示を待たず、自律的に判断・行動する
- 判断基準: 迷ったらシンプルな方を選ぶ
```

### MEMORY.md — 永続記憶

セッションをまたいで保持される記憶です。エージェントが学習したこと、過去の振り返り結果、重要な意思決定の履歴が蓄積されます。PDCAのCheck段階で自動的に更新されます。

### TASKS.md — タスクキュー

実行待ち・実行中・完了のタスクを管理するファイルです。エージェントは起動時にこのファイルを確認し、未完了のタスクから作業を再開します。

### CLAUDE.md — ブート手順

エージェント起動時に最初に実行される手順を定義します。SOUL.md → MEMORY.md → TASKS.mdの読み込み順序、絶対ルール、サブエージェントの呼び出しテンプレートを記述します。

### skills/ — 再利用可能なスキルテンプレート

サブエージェントに渡すプロンプトのテンプレート集です。リサーチ、コーディング、ファイル操作など、頻出するタスクパターンを定義しておくことで、一貫した品質でサブエージェントを起動できます。

## カスタマイズガイド

このブループリントを自分用にカスタマイズする手順です。

1. **SOUL.mdを書き換える** — あなたのエージェントの役割を定義します。「秘書」「リサーチャー」「開発リード」など、ユースケースに応じて自由に設定してください。
2. **CLAUDE.mdのルールを調整** — 絶対ルールやサブエージェントの呼び出しパターンを、あなたのワークフローに合わせて編集します。
3. **skills/にスキルを追加** — 頻繁に使うタスクパターンがあれば、スキルテンプレートとして定義しておくと便利です。
4. **ディレクトリ構造を拡張** — `scripts/`, `data/` など、プロジェクトに必要なディレクトリを追加してください。

> **ポイント:** 最初から完璧を目指す必要はありません。PDCAサイクルにより、エージェント自身がファイルを改善していきます。まずは最小限の設定で動かし、運用しながら育てていくのがおすすめです。
