# スキル: 収益トラッキング

## 概要
副業の各収益チャネルの状況を定期的に確認・記録し、改善提案を行う。

## トラッキング対象

| チャネル | 指標 | 確認頻度 |
|----------|------|----------|
| note | 記事数、ビュー数、スキ数、売上 | 週1回 |
| Zenn | 記事数、Like数、売上 | 週1回 |
| ブログ | PV数、アドセンス収益、アフィリエイト収益 | 週1回 |
| Gumroad | 商品数、売上、ダウンロード数 | 週1回 |
| API/SaaS | MRR、ユーザー数、チャーン率 | 週1回 |

## 週次レポート生成

### サブエージェント指示テンプレート
```bash
claude -p "以下のデータを元に週次収益レポートを生成し、~/agent/logs/revenue_weekly_[日付].md に保存せよ。

## 入力データ
[Junyaから提供された各チャネルの数値]

## レポート内容
1. **サマリー**: 総収益、前週比、目標達成率
2. **チャネル別**: 各チャネルの詳細数値と前週比
3. **トップコンテンツ**: 最も収益に貢献したコンテンツ
4. **改善提案**: データに基づく具体的なアクション3つ
5. **来週の計画**: 優先すべきアクション

## 出力形式
Markdown。数値はテーブル形式。改善提案は具体的に。
" --dangerously-skip-permissions
```

## 月次振り返り
- 月末にPDCAサイクルで収益戦略を見直す
- 伸びているチャネルにリソースを集中
- 伸びないチャネルは撤退基準（3ヶ月で月1万円未満なら見直し）

## 記録ファイル
- 週次: ~/agent/logs/revenue_weekly_[日付].md
- 月次: ~/agent/logs/revenue_monthly_[年月].md
- 累計: ~/agent/logs/revenue_summary.md
