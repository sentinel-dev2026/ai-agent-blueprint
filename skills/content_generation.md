# スキル: コンテンツ自動生成

## 概要
デジタルコンテンツ（有料記事、ブログ記事、電子書籍）のドラフトを自動生成する。

## 対応プラットフォーム
- note（有料記事）
- Zenn（技術記事・本）
- WordPress（ブログ記事）
- Gumroad（デジタル商品）

## 生成フロー

### 1. テーマ選定
- Junyaの専門領域（Java/Spring/AWS/AI）から選定
- SEOリサーチ結果からキーワードを決定
- 競合記事を分析して差別化ポイントを特定

### 2. 構成作成
```
タイトル: [SEOキーワード] + [ベネフィット]
導入: 読者の課題を提示（100〜200字）
本文: 3〜5セクション（各300〜500字）
まとめ: アクションアイテム + CTA
```

### 3. ドラフト生成（サブエージェント指示テンプレート）
```bash
claude -p "以下の仕様でコンテンツを生成し、~/agent/content/drafts/[filename].md に保存せよ。

## 仕様
- テーマ: [テーマ]
- ターゲット読者: [読者像]
- プラットフォーム: [note/Zenn/WordPress]
- 文字数: [目安文字数]
- トーン: 実務経験に基づく、具体例豊富、初心者にもわかりやすい
- SEOキーワード: [キーワード]
- 差別化: 筆者の実体験・具体的な数値を含める

## 制約
- 嘘の体験を書かない（Junyaに確認が必要な箇所は[要確認]と記載）
- AI生成感を減らす（冗長な前置き・まとめを省く）
- 具体的なコード例やスクリーンショット指示を含める
" --dangerously-skip-permissions
```

### 4. レビュー・公開
- Junyaが5〜30分でレビュー
- [要確認]箇所を埋める
- プラットフォームに公開

## 品質チェックリスト
- [ ] タイトルにSEOキーワードが含まれている
- [ ] 導入部で読者の課題が明確に提示されている
- [ ] 具体例・コード例が3つ以上含まれている
- [ ] 独自の視点・体験が含まれている
- [ ] CTAが含まれている（他の記事への誘導、フォロー促進等）
- [ ] 誤情報・嘘の体験が含まれていない

## コンテンツカレンダー管理
- 生成予定: TASKS.mdに記録
- 完成ドラフト: ~/agent/content/drafts/
- 公開済み: ~/agent/content/published/
- 実績: ~/agent/logs/content_log.md
