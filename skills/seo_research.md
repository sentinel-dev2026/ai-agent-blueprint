# スキル: SEO・キーワードリサーチ

## 概要
ブログ記事・コンテンツのテーマ選定のため、検索需要・競合度を調査する。

## 調査フロー

### 1. キーワード候補の洗い出し
```bash
claude -p "以下のジャンルについてSEOキーワードリサーチを行い、結果を ~/agent/logs/seo_research_[日付].md に保存せよ。

## ジャンル: [ジャンル名]

## 調査内容
1. 関連キーワードを20〜30個洗い出す
2. 各キーワードについて以下を調査:
   - 検索意図（情報収集/比較検討/購入）
   - 推定競合度（高/中/低）— 上位10件のドメインパワーから判断
   - コンテンツタイプ（ハウツー/レビュー/まとめ/比較）
   - アフィリエイト可能性（関連商品・サービスの有無）
3. 優先度順にランキング（低競合 × 高需要 × アフィ可能 = 高優先）

## 出力形式
Markdownテーブルで、優先度S/A/B/Cランク付き。
各Sランクキーワードには記事構成案（見出しレベル）も添付。
" --dangerously-skip-permissions
```

### 2. 競合分析
```bash
claude -p "以下のキーワードで上位表示されている記事を分析し、差別化戦略を提案せよ。結果は ~/agent/logs/competitor_[キーワード].md に保存。

## キーワード: [キーワード]

## 分析項目
1. 上位5記事のタイトル・URL・推定文字数
2. 共通して含まれている情報
3. 不足している情報（差別化ポイント）
4. 推奨する記事構成
5. 推奨する差別化要素（実体験、独自データ、コード例等）
" --dangerously-skip-permissions
```

### 3. 定期キーワード更新
- 月1回、トレンドキーワードを再調査
- 既存記事のリライト候補を特定
- 新規キーワードをコンテンツカレンダーに追加

## 注力ジャンル（Junyaの強み）
1. **AWS**: コスト削減、設計パターン、認定資格
2. **Spring Boot**: 実務Tips、AI統合、マイクロサービス
3. **エンジニア副業**: 体験記、ツール紹介、スキル活用法
4. **AI活用**: Claude活用術、開発効率化、自動化
